# PRD：纪律观察室

## 1. 背景

用户需要一个围绕 A股交易纪律的看盘与复盘工具，解决以下问题：

- 早盘信息杂乱，缺少结构化观察清单。
- 晚间复盘容易凭感觉，缺少可沉淀的复盘框架。
- 网络热点与 A股题材、板块、个股之间缺少映射。
- 满仓、单票重仓、热点退潮、利好兑现等风险需要纪律化提醒。

## 2. 产品定位

纪律观察室不是交易软件，不负责下单，不承诺预测准确率。它是一个观察、记录、复盘、纪律提醒工具。

## 3. 目标用户

- 关注 A股短线/波段的个人投资者。
- 需要早盘计划与晚间复盘模板的人。
- 需要追踪网络热点并映射到 A股题材的人。
- 需要控制仓位和减少情绪化交易的人。

## 4. 核心使用流程

### 4.1 早盘流程

1. 导入昨夜外围、指数、板块、热点、自选股数据。
2. 系统生成早盘观察清单。
3. 用户确认今日主线、风险点和纪律要求。
4. 输出早盘 Markdown 报告。

### 4.2 盘中流程，MVP 可弱化

1. 查看热点变化。
2. 查看自选股风险提醒。
3. 人工记录关键事件。

### 4.3 晚间复盘流程

1. 导入当日市场数据。
2. 对照早盘计划。
3. 记录执行偏差。
4. 生成复盘报告。
5. 沉淀明日观察重点。

## 5. MVP 功能

### 5.1 Dashboard

- 市场状态卡片。
- 热点强度 Top 10。
- 自选股风险提醒。
- 最新报告入口。

### 5.2 自选股管理

字段：

- 股票代码
- 股票名称
- 所属板块
- 关联概念
- 当前价
- 成本价
- 仓位比例
- 止损价
- 目标价
- 持仓状态
- 备注

### 5.3 网络热点追踪

字段：

- 平台
- 关键词
- 排名
- 热度值
- 情绪
- 摘要
- 关联概念
- 关联个股
- 采集时间

### 5.4 热点映射

- 关键词到概念。
- 概念到板块。
- 概念到个股。
- 支持手动编辑映射。

### 5.5 风控提醒

风险提醒包括：

- 满仓。
- 单票过重。
- 跌破止损。
- 利好兑现。
- 热点退潮。
- 情绪过热。

### 5.6 报告中心

- 查看报告列表。
- 生成报告。
- 复制 Markdown。
- 导出 HTML，后续可扩展 PDF。

## 6. 非功能需求

- 本地运行优先。
- 操作简单。
- 数据结构清晰。
- 易扩展真实数据源。
- 所有报告可追溯。
- 关键规则可配置。

## 7. 合规要求

- 不做自动下单。
- 不抓取需授权或禁止采集的平台数据。
- 不输出确定性收益承诺。
- 不把工具包装成荐股系统。
- 所有提示均以“观察、风险、纪律”为表达核心。

## 8. 后续版本方向

- 接入合规行情 API。
- 接入新闻聚合 API。
- 增加多账户复盘。
- 增加交易日志统计。

- 增加热点生命周期曲线。
- 增加移动端适配。
